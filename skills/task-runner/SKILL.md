# Task Runner - 连续任务执行器

## 核心机制

```
对话 ➜ 读取 task.md ➜ 🔄 重载 SKILL.md ➜ 执行第一个 [] ➜ 标记 [x] ➜ ⚠️ 重新读取磁盘文件 ➜ 循环
```

## 文件路径

以下路径为本 skill 的唯一权威路径，执行时严格以此为准：
PATH:
    ../task.md

## ⚠️ 关键规则（必须严格遵守）

### 🚨 规则0：对话输出优先（最高优先级）

> **每个任务的执行结果必须先以完整内容输出到对话中，然后才能写入 task.md 归档。**

**判定标准**：

1. 必须包含对任务问题的**直接回答**——不是进度通知
2. 解读/分析/设计类任务：需包含结构化内容，深度与问题复杂度匹配
3. **质量等价**：每个任务的回答质量等同于用户单独提问，不因批量执行而缩减

```
🚨 严重违规（最常见错误）：
  "我已经完成了第N项任务" ← 这是进度通知，不是结果
  "已完成分析，正在归档"  ← 同上，用户什么都没看到

✅ 正确做法：
  先输出完整实质内容（分析、结论、代码等）→ 再写入 task.md 归档摘要
```

**执行检查点**：写入 `[x]` 之前回顾本轮对话——若未输出该任务的实质内容，先补输出再归档。

### 规则1：动态任务检查

> 每完成一个任务后，必须 `read_file(task.md)` 重新从磁盘读取。禁止依赖内存缓存判断是否还有未完成任务。

```
❌ 错误：首次读取 → 遍历内存列表 → 全部完成 → 归档
✅ 正确：完成N → read_file → 扫描 [] → 发现N+1 → 执行 → read_file → 循环
```

### 规则2：归档

每完成一个任务后，在标记 `[x]` 的**原任务标题下方**直接追加归档内容：

```
## [x] <原任务标题>

**完成内容**
- ✅ 完成点1：...
- ✅ 完成点2：...
```

- 每完成一个任务就立即写入，禁止在最后一次性补写。
- 归档直接覆盖写入 task.md，无需征求用户确认。

### 规则3：顺序执行

按文件中出现顺序（从上到下）逐个执行。无明确指令时，不跳过、不并行、不重排。

### 规则4：任务识别

仅识别 Markdown checklist 格式：`[]`（未完成）/ `[x]`（已完成）。普通段落、说明文字、标题不算任务。

### 规则4.5：规则热重载（防止规则遗忘）

在以下时机，必须先 `read_file(SKILL.md)` 重新载入规则，再继续执行：
- **执行循环中每次即将执行下一个 `[]` 任务前**（步骤 2）
- 进入规则5等待态之前
- `askQuestions` 返回后、执行用户选择之前

目的：防止多轮循环后因上下文压缩而遗忘规则。选项文本中的提醒仅为辅助，不能替代重载。

### 规则5：任务流结束策略（防止流程直接终止）

当扫描未发现任何 `[]` 时，**禁止直接结束**，必须调用 `askQuestions` 进入等待态。

**问题文本**：`当前无待办任务，请选择下一步操作：`（含默认提醒 "先重新遵守 task-runner 全部规则"）

| 选项 | 行为 |
|------|------|
| **继续执行** | 立刻 `read_file(task.md)` 重新扫描 |
| **直接输入任务描述** | 新增任务入口（见规则6） |
| **暂不继续** | 直接结束对话，不再输出任何内容 |

- 选项 description 建议包含"将先重载 SKILL.md 再继续执行"。
- **异常处理**：`askQuestions` 返回 skipped / 空输入 / 无法映射 / 调用失败时，重新调用 `askQuestions` 重试。

### 规则6：新增任务入口

用户在 `askQuestions` 中选择"直接输入任务描述"并提交文本后：

1. 将输入按 checklist 格式追加到 task.md 末尾：`## [] <任务描述>`
2. 不改动已完成任务的归档块。
3. 写入后立刻 `read_file(task.md)`，返回执行循环步骤 2。
4. 输入为空/仅空白时，提示重新输入，不写入文件。

## 执行循环

1. `read_file(task.md)`
2. `read_file(SKILL.md)` 重载规则（规则4.5）
3. 扫描第一个 `[]` 未完成项
4. 执行任务，**在对话中输出完整结果**（规则0）——把每个 `[]` 当作独立提问
5. 标记 `[x]` + 追加归档摘要（规则2）
6. `read_file(task.md)` 获取磁盘最新版
7. 重复 2-6 直到无 `[]`（每次循环都经过步骤 2 重载规则）
8. `read_file(SKILL.md)` 重载规则 → `askQuestions` 进入等待态（规则5）
9. 用户返回后：
   - **继续执行** → `read_file(SKILL.md)` → 回步骤 1
   - **直接输入任务描述** → `read_file(SKILL.md)` → 规则6 写入 → 回步骤 1
   - **暂不继续** → 结束对话
