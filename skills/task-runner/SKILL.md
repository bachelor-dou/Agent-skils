# Task Runner - 连续任务执行器

## 核心机制

```
对话 ➜ 读取 task.md ➜ 执行第一个 [] ➜ 标记 [x] ➜ ⚠️ 重新读取磁盘文件 ➜ 查找下一个 [] ➜ 循环
```

## 关键文件路径声明

声明当前skill中涉及的文件路径，执行时必须以此处路径为准：
PATH:
    ../task.md

## ⚠️ 关键规则（必须严格遵守）

### 🚨 规则0：对话输出优先（所有规则中最高优先级）

> **每个任务的执行结果必须先以完整内容输出到对话中，然后才能写入 task.md 归档。**

**判定标准**：

1. 包含对任务问题的**直接回答**（不是"已完成"、"已分析"等进度通知）
2. 解读/分析/设计类：需包含结构化内容，篇幅与问题复杂度匹配
3. **质量等价**：回答深度等同于用户单独提问时的质量，不因批量执行而缩减
4. 回答应自然完整地讲清楚核心内容，用最适合当前问题的方式组织

```
🚨 严重违规（最常见错误）：
  "我已经完成了第N项任务，接下来会把它标记为已完成" ← 这不是结果，这是进度通知
  "已完成该任务的分析，正在写入归档" ← 同上，用户什么都没看到

✅ 正确做法：
  先输出任务的完整实质内容（分析、结论、代码等）→ 用户看到完整内容后 → 再写入 task.md 归档摘要
```

**执行检查点**：在调用 replace_string_in_file 写入 task.md `[x]` 标记之前，回顾本轮对话——如果对话中没有输出该任务的实质内容，必须先补输出，再写归档。

### 规则1：动态任务检查

> 每次完成一个任务后，必须使用 `read_file` 工具重新从磁盘读取 task.md。禁止依赖内存缓存判断"是否还有未完成任务"。

```
❌ 错误：首次读取 ➜ 遍历内存列表 ➜ 全部完成 ➜ 归档
✅ 正确：完成任务N ➜ read_file(task.md) ➜ 扫描 [] ➜ 发现N+1 ➜ 执行 ➜ read_file(task.md) ➜ 循环
```

### 规则2：归档

每完成一个任务后，执行归档操作：

**归档（写入 task.md）**

在已标记 `[x]` 的**原任务标题下方**直接追加归档内容，禁止另起 `## [x] 任务N` 等独立标题。格式如下：

```
## [x] <原任务标题>

**完成内容**
- ✅ 完成点1：...
- ✅ 完成点2：...
- ✅ 完成点3：...
```

- 每完成一个任务就立即写入，禁止只在最后一次性补写。
- 归档与状态更新对 `task.md` 直接覆盖写入，无需额外征求用户确认。

### 规则3：顺序执行

- 任务必须按文件中出现顺序执行（从上到下）。
- 无明确指令时，不跳过、不并行、不重排。

### 规则4：任务识别

- 仅识别 Markdown checklist：`[]` / `[x]`。
- 普通段落、说明文字、标题不算任务。

### 规则4.5：Skill 热重载（防上下文压缩）

- 在以下时机，必须先使用 `read_file` 重新读取当前 `SKILL.md`，再继续执行：
  - 进入规则5等待态之前
  - `askQuestions` 返回“继续执行”之后
  - `askQuestions` 返回“直接输入任务描述”之后
- 这一步的目的，是把 task-runner 的完整规则重新载入当前轮上下文，防止多轮循环后因上下文压缩而遗忘规则。
- 禁止仅依赖选项文本中的提醒语替代这一步；选项提醒只能作为辅助，不能代替重载。

### 规则5：任务流结束策略（重要: 防止流程直接终止）

当扫描未发现任何 `[]` 时，不允许直接结束，必须调用 `askQuestions` 进入等待态，提示用户以下选项：

- **继续执行**：立刻 `read_file(task.md)` 重新扫描。
- **直接输入任务描述**：新增任务入口（见规则6）。
- **暂不继续**：**直接结束本轮对话，不再输出任何内容。**

调用 `askQuestions` 时，question 必须显式包含默认提醒，建议使用以下模板：

``` 
先重新遵守 task-runner 全部规则，再选择下一步。当前无待办任务，你要做什么？
```

- 这段提醒属于等待态默认 prompt，用户无需手动再输入“注意遵守 skill 规则”。
- 选项 description 也应尽量包含“将先重载 SKILL.md 再继续执行”的提示语。

**异常处理**：若 `askQuestions` 返回 skipped、空输入、无法映射到上述三类选项、或调用失败时，再次调用 `askQuestions` 重试.

### 规则6：新增任务入口（askQuestions 直接录入）

当用户在 `askQuestions` 中选择"直接输入任务描述"并提交文本时，必须执行以下步骤：

1. 将用户输入的任务描述按 task.md checklist 格式直接写入 task.md：

```
## [] <用户输入的任务描述>
```

2. 写入位置：追加到 task.md 末尾，不改动已有已完成任务归档块。
3. 写入后立刻 `read_file(task.md)`，并返回执行循环步骤 2 开始处理新任务。
4. 如果输入为空或仅空白，提示用户重新输入有效任务描述，不写入文件。

## 执行循环（必须严格按以下步骤循环执行）

1. `read_file(task.md)`。
2. 扫描第一个未完成项 `[]`。
3. 执行该任务，**在对话中向用户输出完整结果**（规则0）。把每个 `[]` 任务当作用户的一次独立提问来回答，不因批量执行而缩减质量。
4. 将该任务标记为 `[x]` 并追加归档摘要（规则2）。
5. 立刻再次 `read_file(task.md)` 读取磁盘最新版本。
6. 重复步骤 2-5，直到无 `[]`。
7. 先 `read_file(SKILL.md)` 重载当前 skill 规则，再调用 `askQuestions` 进入等待态（规则5）；若返回异常按规则5异常处理段重试。
8. 若用户选择继续执行，先 `read_file(SKILL.md)`，再返回步骤 1。(必须严格执行)
9. 若用户直接输入任务描述，先 `read_file(SKILL.md)`，按规则6写入 task.md 后，再返回步骤 1。(必须严格执行)
10. 若用户选择暂不继续，直接结束本轮对话，不再输出任何内容。
